Engineering Dashboard — environment setup
==========================================
Set these in Vercel → your project → Settings → Environment Variables, then Redeploy.

------------------------------------------------------------------
1) SHARED DATABASE (so dashboard edits persist and are shared)  ✅ DONE
------------------------------------------------------------------
Vercel → Storage → Create Database → "Upstash for Redis", connect to the project.
Vercel auto-adds KV_REST_API_URL and KV_REST_API_TOKEN.

------------------------------------------------------------------
2) DAILY EMAIL — sending via Gmail / Google Workspace (SMTP)
------------------------------------------------------------------
Chosen setup: send FROM and TO ashwinee@actiontourguide.com (temporary).
No DNS needed. On the sending Google account (ashwinee@actiontourguide.com):
  a) Turn on 2-Step Verification (Google Account → Security).
  b) Create an App Password (Security → App passwords) — a 16-character code.

Then set these env vars in Vercel:
  SMTP_USER    ashwinee@actiontourguide.com
  SMTP_PASS    the 16-character app password (paste without spaces)
  REPORT_FROM  Engineering Dashboard <ashwinee@actiontourguide.com>
  REPORT_TO    ashwinee@actiontourguide.com
  CRON_SECRET  any long random string (required; also used to protect the endpoint)
  # optional: SMTP_HOST (default smtp.gmail.com), SMTP_PORT (default 465)

(Alternative provider: set RESEND_API_KEY instead of SMTP_* to use Resend.)

The cron in vercel.json calls /api/report every day at 1:00 PM IST (07:30 UTC).

Manual test after setting variables + redeploying:
  https://<your-app>.vercel.app/api/report?key=<CRON_SECRET>
A 200 {"ok":true,"via":"smtp",...} means the email was sent with live data.
